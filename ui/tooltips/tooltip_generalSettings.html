Show Tooltips: Shows all the tooltips.

Hide Guide after Rig Build: This will hide the guide when a rig build is done.

Hide Rig after Guide Build: This will hide the rig when a guide build is done.

Show Guide after Guide Build: This will show the guide when a guide build is done.

Show Rig after Rig Build: This will show the rig when a rig build is done.

Pop-up Guide Properties: When selecting a guide component group, the Guide Settings will pop up automatically.

Remove Reference after Build: Will remove the reference after a build process. The build data is stored in maya files which are being referenced when building them, for instance guides or deformation. Keeping the reference in the scene after building will allow you to do custom transfers to your rig if needed, for example when you want to transfer weights from or to specific joints using joint labeling. In this case you can modify the joint labels in the referenced rig to make custom skin weight transfers.

Use Custom Delivery Saving: You have the option to use a custom saving method for the Save Delivery option in the File menu. Once activated, the system will run the script defined in Delivery Script under the Global Scripts rollout from the BEAR Builder Settings.

Create Controller Tags: Creates controller tags for all controls on rig build. This setting may impact performance.

Version Naming: The naming convention for the version token. v*** will return the token v001, v002 etc. for the version folder.

Maya File Type: Binary files are recommended as they save and load faster. This can make a significant difference when working with complex rigs and large skin or blendshape data.
